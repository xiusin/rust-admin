# 电商SaaS平台 - 技术开发设计文档

> **基于 gin-vue-admin 框架 | PostgreSQL | 多商户SaaS架构**  
> **文档版本**: v1.0 | **撰写日期**: 2026-03-05  
> **竞品参考**: 诺云直播SaaS平台 (nuoyun.shumeiai.cn)

## 一、架构总览

### 1.1 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Go + Gin | 1.23 / 1.10.0 |
| ORM | GORM | 1.25.12 |
| 数据库 | PostgreSQL | - |
| 缓存 | Redis | 9.7.0 |
| 前端框架 | Vue 3 + Composition API | 3.5.7 |
| UI组件库 | Element Plus | 2.10.2 |
| CSS框架 | UnoCSS | 66.4.2 |
| 构建工具 | Vite | 6.2.3 |
| 状态管理 | Pinia | 2.2.2 |
| 图表 | ECharts | 5.5.1 |
| 支付 | 易宝支付 + 微信支付 | - |
| 对象存储 | 阿里云OSS / MinIO / 本地 | - |

### 1.2 SaaS 多商户架构

采用 **共享数据库 + 共享Schema + tenant_id隔离** 模式。

- 新增 `shop_tenant` 商户表，每个商户一条记录
- 所有业务表(shop_*前缀)含 `tenant_id` 字段
- Service层统一注入 tenant_id 过滤
- 通过GVA现有JWT + Authority体系绑定商户

### 1.3 模块依赖关系

- **字典管理** → 为全模块提供枚举值
- **商户管理** → SaaS基座，所有业务模块依赖
- **素材管理** → 为商品/门户等模块提供图片视频
- **商品管理** → 依赖分类、素材、运费模板
- **用户管理** → C端用户、会员、标签体系
- **订单管理** → 依赖商品SKU、用户、支付、物流
- **支付管理** → 订单支付、退款通道
- **物流管理** → 发货关联、运费计算
- **数据统计** → 聚合订单/商品/用户数据

## 二、GVA代码组织结构

### 2.1 后端新增目录

所有电商模块统一放在 `shop` 命名空间下:

- `server/api/v1/shop/` - API控制器层
- `server/model/shop/` - 数据模型层(含request/response子目录)
- `server/service/shop/` - 业务逻辑层
- `server/router/shop/` - 路由注册层

每个层级都有 `enter.go` 统一注册。

### 2.2 前端新增目录

- `web/src/api/shop/` - API接口定义
- `web/src/view/shop/` - 页面组件(按模块分子目录)

### 2.3 enter.go 注册示例

**service/shop/enter.go**:
```go
package shop

type ServiceGroup struct {
    TenantService
    ProductService
    ProductCategoryService
    ProductGroupService
    ProductSkuService
    OrderService
    OrderRefundService
    OrderShippingService
    MaterialService
    MaterialCategoryService
    LogisticsService
    ShippingTemplateService
    PaymentService
    AccountService
    CustomerService
    CustomerTagService
    MemberLevelService
}
```

**api/v1/shop/enter.go**:
```go
package shop

import "github.com/flipped-aurora/gin-vue-admin/server/service"

type ApiGroup struct {
    TenantApi
    ProductApi
    ProductCategoryApi
    ProductGroupApi
    OrderApi
    OrderRefundApi
    OrderShippingApi
    MaterialApi
    MaterialCategoryApi
    LogisticsApi
    ShippingTemplateApi
    PaymentApi
    AccountApi
    CustomerApi
    CustomerTagApi
    MemberLevelApi
}

var shopService = service.ServiceGroupApp.ShopServiceGroup
```

在顶层 `service/enter.go`、`api/v1/enter.go`、`router/enter.go` 中新增 `ShopServiceGroup`、`ShopApiGroup`、`ShopRouterGroup` 注册。

## 五、核心业务流程

### 5.1 商品生命周期

```
创建商品(草稿) → 提交审核 → 审核通过 → 上架
                           → 审核不通过 → 修改 → 重新提交
上架 ⇄ 下架
下架 → 删除(回收站) → 30天自动清理/恢复
```

### 5.2 多规格SKU设计流程

```
1. 创建商品基本信息(SPU级)
2. 选择"多规格" → 添加规格名(如"颜色","尺码")
3. 为每个规格名添加规格值(如"红色","蓝色","S","M","L")
4. 系统自动生成SKU笛卡尔积组合:
   红色-S, 红色-M, 红色-L, 蓝色-S, 蓝色-M, 蓝色-L
5. 为每个SKU设置独立的: 售价、划线价、成本价、库存、图片
6. SPU的 sale_price = MIN(所有SKU售价), stock = SUM(所有SKU库存)
```

### 5.3 订单状态流转

```
买家下单 → 待付款 ──[30分钟超时]──→ 已关闭(库存回补)
              │
              ▼ 付款成功
           待发货 ──→ 商家发货 ──→ 待收货
              │                      │
              │                      ├─ [7天自动]确认收货 ──→ 已完成
              │                      │
              │                      └─ 可申请延长收货(1次,+7天)
              │
              └─ [拼团订单] ──→ 待成团 ──→ 成团成功 → 待发货
                                       └→ 成团失败 → 自动退款 → 已关闭
```

### 5.4 退款状态流转

```
买家申请退款 → 待商家处理 ──[7天超时]──→ 自动退款
                   │
                   ├─ 同意退款
                   │     ├─ 仅退款 → 退款成功
                   │     └─ 退货退款 → 待买家寄回 → 买家填物流 → 待商家收货 → 确认收货 → 退款成功
                   │
                   └─ 拒绝退款 → 待买家处理
                                    ├─ 修改申请 → 待商家处理
                                    ├─ 撤销申请 → 退款关闭
                                    └─ 申请平台介入 → 平台裁决
```

### 5.5 支付流程(易宝支付为主)

```
1. 买家确认订单 → 前端调用 创建支付接口
2. 后端生成订单号 → 调用易宝支付API创建支付单
3. 返回支付参数(H5/小程序/扫码) → 前端唤起支付
4. 用户完成支付 → 易宝回调通知
5. 后端验签 → 更新订单状态(已支付) → 记录支付流水
6. 推送通知给买家(订单已支付)
7. 退款: 调用易宝退款API → 等待回调 → 更新退款状态
```

### 5.6 库存管理规则

```
下单扣库存模式:
- 创建订单时立即锁定库存 (UPDATE SET stock = stock - N WHERE stock >= N)
- 并发防超卖: 使用数据库行锁 + 乐观锁
- 未付款超时关闭: 库存自动回补
- 退款成功: 库存自动回补
- 库存为0: 商品标记售罄
```

## 十、安全与性能设计

### 10.1 安全设计

- 支付密钥AES加密存储，运行时解密
- 支付回调接口验签防篡改
- 订单号使用雪花算法，不可推测
- 库存扣减使用数据库行锁防超卖
- C端用户手机号脱敏展示(138****8888)
- 操作日志全记录(使用GVA OperationRecord)

### 10.2 性能设计

- 商品列表查询走Redis缓存(TTL 5min)
- 订单创建使用分布式锁(Redis)防重复提交
- 素材上传使用分片上传+断点续传
- 数据统计走定时任务预聚合(非实时查询)
- 数据库索引已在ER设计中规划
- 大表查询强制走索引(tenant_id + 状态字段)
